% ABSTRACT
% ============================================================
\begin{abstract}
In pipelined processors, every mispredicted branch forces a pipeline flush,
discarding prefetched instructions and directly reducing throughput. For a
5-stage RISC-V pipeline, each misprediction costs two penalty cycles. The
standard always-not-taken static predictor requires no hardware but achieves
only 27.50\% accuracy on branch-intensive workloads---an outcome that is
structurally unavoidable. Fully neural predictors improve accuracy but activate
expensive dot-product inference on every branch, including easy high-bias cases
where a simpler predictor already succeeds. Neither extreme is well-suited for
resource-constrained embedded or FPGA platforms.

This work presents an FPGA-validated 5-stage RISC-V RV32I SoC
integrating a \emph{confidence-adaptive hybrid branch predictor}. The predictor
applies a bimodal saturating-counter gate as a fast confidence screener: only
branches flagged as low-confidence are escalated to the perceptron neural
predictor. Within the perceptron path, a hardware early-stopping rule in the combinational
adder tree halts accumulation as soon as the partial weighted sum makes the
prediction outcome mathematically certain, reducing unnecessary switching
activity. A 1-cycle output register enables full timing closure at 125\,MHz.
The entire arithmetic implementation uses 4-bit integer operations with no
multiply-accumulate blocks, making the design viable for entry-level FPGA
fabric and standard-cell ASIC flows.

The system is implemented in synthesizable Verilog RTL and validated on the
PYNQ-Z2 FPGA platform (Zynq-7000, 125\,MHz). Across a six-program evaluation
suite spanning strongly-biased loops, nested loops, alternating patterns, and
mixed-bias workloads, the hybrid predictor achieves an average accuracy of
\textbf{84.39\%} versus \textbf{23.87\%} for the always-not-taken baseline,
delivering an average \textbf{78.31\% reduction} in misprediction penalty
cycles. On the primary 40-branch stress program, accuracy reaches 75.00\%
with a 65.52\% penalty reduction; on a 32-branch strongly-biased loop, accuracy
reaches 96.88\% with a 96.77\% penalty reduction. Confidence gating limits
neural activation to 55\% of branches on the stress program. Both CPU
variants achieve full place-and-route timing closure at 125\,MHz on the
PYNQ-Z2 (Zynq-7000, WNS\,=\,+0.006\,ns), enabled by a pipelined
parallel-tree adder in the perceptron inference path.
\end{abstract}

\section{Concept Description}
\label{sec:concept}

\subsection{Background and Motivation}
\label{sec:background}

Pipelined processors overlap the execution of multiple instructions to improve
throughput. The standard 5-stage pipeline---Instruction Fetch (IF), Instruction
Decode (ID), Execute (EX), Memory Access (MEM), and Write-Back (WB)---is
the foundational microarchitecture for balancing implementation complexity against
instruction-level parallelism~\cite{patterson2020cod,hennessy2017quantitative}.
The critical challenge arises at conditional branch instructions: the branch
outcome cannot be determined until the EX stage, two cycles after the branch
enters the pipeline. If the processor waits, two cycles are wasted. If it
speculatively prefetches the wrong path and must flush, two partially-executed
instructions are discarded---the \emph{branch misprediction penalty}.

For a processor targeting embedded or edge applications, this penalty directly
limits effective throughput. The intuitive solution---a branch predictor---must
be evaluated on three axes simultaneously: accuracy, hardware cost, and
integration complexity. Static always-not-taken prediction costs nothing but
fails on loop-dominated workloads. Full perceptron prediction offers accuracy
gains but introduces neural inference overhead on every branch regardless of
difficulty. The engineering opportunity is a \emph{selective} hybrid that applies
neural computation only where it is needed, driven by real-time confidence
measurement---a design that this project realizes and validates at the
register-transfer level.

\subsection{Problem Statement}
\label{sec:problem}

Three concrete limitations motivate this work:

\begin{enumerate}
  \item \textbf{Static predictor accuracy ceiling.}
    The always-not-taken baseline achieves 27.50\% accuracy on the branch-stress
    evaluation program (11 correct out of 40 conditional branch resolutions,
    yielding 29 mispredictions and approximately 58 penalty cycles). This is
    structurally unavoidable: the program's 20-iteration loop with XOR-toggled
    conditions produces mostly taken branches, which a fixed not-taken policy
    cannot capture at all.

  \item \textbf{Resource cost of always-on neural inference.}
    A perceptron predictor that activates on every branch executes an 8-step
    dot-product regardless of whether the branch is already predictable by a
    simpler method. On entry-level FPGAs without dedicated DSP blocks, this
    wastes LUT switching activity and complicates timing closure. The hardware
    cost must be proportional to the difficulty of the branch, not applied
    uniformly.

  \item \textbf{No hardware-optimized hybrid framework for RV32I.}
    Published perceptron predictor designs are primarily standalone components
    described at the algorithmic level~\cite{jimenez2001perceptron,jimenez2003fast}.
    Integrating a confidence-adaptive hybrid predictor into the timing-critical
    ID stage of an RV32I 5-stage pipeline---with sideband registers preserving
    prediction metadata for precise EX-stage update semantics---requires a
    custom architecture co-design that has no reference implementation in the
    open-source RISC-V ecosystem.
\end{enumerate}

\subsection{Objectives and Expected Contributions}
\label{sec:objectives}

This project achieves the following concrete objectives:

\begin{enumerate}
  \item Deliver a synthesizable 5-stage pipelined RISC-V RV32I SoC in
    Verilog RTL with a lightweight ISA subset (23 of 40 base instructions),
    targeting the operations needed for branch predictor evaluation while
    keeping area minimal, with a hazard detection unit handling load-use stalls
    and control hazard flushes~\cite{waterman2016riscv,palnitkar2003verilog}.

  \item Design and implement a confidence-adaptive hybrid branch predictor
    combining a bimodal 2-bit saturating counter~\cite{smith1981study} with
    a perceptron neural predictor~\cite{jimenez2001perceptron}, activating the
    neural path only on low-confidence branches.

  \item Implement a hardware early-stopping rule within the perceptron combinational
    inference path that reduces unnecessary switching activity, paired with a
    1-cycle output register to achieve timing closure at 125\,MHz.

  \item Integrate the predictor at the ID stage using sideband pipeline registers
    that carry prediction metadata (predicted outcome, bimodal state, GHR snapshot,
    neural-path flag) in lock-step with the instruction flow, enabling precise
    EX-stage update and rollback semantics.

  \item Validate the complete system on the PYNQ-Z2 FPGA at 125\,MHz and
    quantify prediction improvement against a directly comparable baseline CPU
    through a self-checking, head-to-head testbench suite.
\end{enumerate}

\section{Work Design}
\label{sec:design}

\subsection{Overall System Concept}
\label{sec:system-concept}

The proposed system is a single-chip SoC consisting of three top-level components:
the pipelined CPU core with integrated branch predictor, a single-cycle instruction
ROM, and a word-addressed data RAM, all unified under one top-level design.
Fig.~\ref{fig:arch} illustrates the complete system hierarchy.

\begin{figure}[htbp]
  \centering
  \includegraphics[width=\linewidth]{figures/fig-arch.pdf}
  \caption{Top-level architecture of the proposed SoC.
           The five pipeline stages (IF--ID--EX--MEM--WB) are shown with
           inter-stage pipeline registers (gray).
           The branch predictor subsystem and its four components attach at
           the ID stage; the hazard control unit manages stall and flush
           signals. Solid arrows indicate data flow; dashed arrows indicate
           control or update paths.}
  \label{fig:arch}
\end{figure}

The CPU core implements a lightweight subset of the RV32I base integer
instruction set, deliberately scoped to the operations required by the branch
predictor evaluation workloads while minimising area on resource-constrained
FPGA targets. The current subset covers 23 instructions: six R-type arithmetic
operations (ADD, SUB, AND, OR, XOR, SLT), six I-type immediates (ADDI, SLTI,
ORI, ANDI, XORI, SLLI), all five standard loads (LB, LH, LW, LBU, LHU),
three stores (SB, SH, SW), two conditional branches (BEQ, BNE), and the
unconditional jump JAL. Remaining RV32I operations---right shifts (SRL, SRA,
SRLI, SRAI), unsigned comparisons (SLTU, SLTIU), additional branches (BLT,
BGE, BLTU, BGEU), indirect jump (JALR), upper-immediate loads (LUI, AUIPC),
and system instructions (ECALL, EBREAK, FENCE)---are architecturally reserved
in the decoder definitions and can be enabled incrementally as application
requirements expand. Peripheral subsystems (UART, SPI, DMA, AXI-Lite bus
interconnect, PLIC/CLINT interrupt controller) are likewise planned for
subsequent milestones.

The branch predictor is designed as a self-contained plugin that interfaces with
the pipeline through two clean ports: (a)~a predict port receiving the ID-stage
PC and branch-type flag, and (b)~an update port receiving the EX-stage actual
outcome, mispredict flag, and GHR snapshot. This decoupled interface allows the
base pipeline and predictor to be synthesized and tested independently, and
simplifies future predictor replacement or parameter tuning without touching the
core datapath.

\subsection{Core Innovation}
\label{sec:innovation}

The central innovation is a \emph{confidence-adaptive two-layer prediction chain}
that applies neural computation selectively and terminates early when possible,
as illustrated in Fig.~\ref{fig:bp-chain}. Four engineering contributions define
this design:

\begin{figure}[htbp]
  \centering
  \includegraphics[width=0.60\linewidth]{figures/fig-bp-chain.pdf}
  \caption{Branch prediction decision chain, spanning the ID stage (blue)
           and EX-stage feedback (orange). Bimodal confidence gating
           bypasses the perceptron for easy branches. The early-stop rule
           $|s_i|>(n-i)\,W_{\max}$ terminates dot-product computation
           once the result sign is determined. Mispredictions trigger both
           pipeline flushing and online weight updates.}
  \label{fig:bp-chain}
\end{figure}

\paragraph{(a) Selective Neural Activation.}
Rather than routing every branch through full neural inference, the predictor
uses a bimodal 2-bit saturating counter as a confidence gate. Only branches in
a \emph{weak counter state} (neither strongly taken nor strongly not-taken)---
those that have recently changed direction and are genuinely hard to predict---
are escalated to the perceptron. This confidence gate limits neural activation
to 55\% of branches on the evaluated workload, leaving 45\% resolved entirely
by the lightweight bimodal path. The consequence is a system where hardware cost
scales with branch difficulty, not with branch frequency.

\paragraph{(b) Hardware-Efficient Early Stopping.}
Within the neural path, inference does not always require all 8 history steps.
After accumulating $i$ terms, if the magnitude of the partial sum $s_i$ exceeds
the maximum possible contribution from all remaining history bits:
\[
  |s_i| \;>\; (n - 1 - i)\,\times\,W_{\max}
\]
the outcome sign is already decided and inference halts immediately. This
condition is evaluated by a purely combinational comparator inside the perceptron
unit~\cite{jimenez2003fast}. In the current pipelined implementation, the
combinational adder tree and early-stop comparator feed a 1-cycle output
register (inserted to achieve timing closure), so the prediction result is
available one cycle after the ID-stage query. The early-stop mechanism reduces
unnecessary switching activity within the combinational path even though the
register boundary fixes the observable output latency at one cycle.

\paragraph{(c) Pipeline-Ready ID-Stage Integration.}
The predictor produces its output within the ID stage, enabling speculative
PC redirection before the EX stage resolves the branch. Dedicated sideband
pipeline registers carry prediction metadata (predicted outcome, bimodal state,
GHR snapshot, neural-path flag) from ID to EX in lock-step with the main
instruction flow. This design ensures that: (i)~the predictor's decision is
available early enough to redirect the fetch stream, and (ii)~the EX-stage
update logic receives the exact history state that was active at prediction
time, guaranteeing correct online learning even in the presence of pipeline
flushes and stalls.

\paragraph{(d) FPGA-Validated Complete Prototype.}
The system is not a simulation-only design. Both the baseline CPU and the
innovation CPU achieve full place-and-route timing closure on the PYNQ-Z2
(Zynq-7000, 125\,MHz, WNS\,=\,+0.006\,ns, zero setup and hold violations).
The entire arithmetic implementation uses 4-bit signed integer add/subtract
and magnitude comparison, consuming only LUT fabric with no DSP block
dependencies. This makes the design directly portable to entry-level FPGAs
and standard-cell ASIC flows.

\paragraph{Versus lightweight educational RISC-V CPUs.}
Widely used open educational RISC-V cores (such as those in introductory
architecture courses) implement always-not-taken or always-taken static
prediction. These designs prioritize simplicity and transparency but provide no
adaptive learning. The proposed design integrates a fully operational learning
predictor whose weights adapt at runtime to observed branch patterns without any
external software support, while preserving the same clean 5-stage pipeline
structure familiar to students and engineers.

\paragraph{Versus commercial processor IP.}
Commercial soft-core IP (e.g., ARM Cortex-M series, SiFive E-series) embeds
proprietary predictors unavailable for inspection or modification. The proposed
RTL is fully open, parameterized, and designed for study, experimentation, and
extension. It provides a transparent, hardware-verified alternative for academic
research and FPGA-based soft-core prototyping.

\subsection{Target Application Domains}
\label{sec:domains}

\begin{itemize}
  \item \textbf{Edge controllers and IoT SoCs.}
    Resource-constrained RISC-V microcontroller designs for sensor fusion,
    motor control, and real-time event processing benefit directly from a
    predictor that concentrates neural computation on hard branches. The
    confidence gate is designed to reduce unnecessary neural toggling on the
    majority of branches, which is valuable in battery-powered or duty-cycled
    edge nodes.

  \item \textbf{FPGA soft-core platforms.}
    FPGA designs that require an embedded programmable processor---for custom
    communication protocols, hardware accelerator control, or reconfigurable
    control planes---can adopt this design as a drop-in soft-core. PYNQ-Z2
    validation at 125\,MHz confirms immediate usability on the Zynq-7000
    device family, and the synthesizable coding style ensures portability to
    other Xilinx and Intel FPGA targets.

  \item \textbf{Branch-intensive embedded workloads.}
    Firmware for digital signal processing, finite-state-machine-driven protocol
    stacks, and real-time control loops exhibits dense, repetitive branch
    patterns---exactly the regime where GHR-based perceptron weights converge
    fastest. The proposed predictor is architecturally well-matched to these
    workloads.

  \item \textbf{Pre-silicon ASIC prototyping and architecture research.}
    The parameterized predictor design, self-checking testbench suite, and
    complete Vivado synthesis flow make this system a ready-to-use evaluation
    vehicle for branch-predictor research prior to ASIC tape-out. The absence
    of FPGA-specific primitives and DSP macros ensures direct portability to
    standard-cell flows.
\end{itemize}

\section{Implementation Method}
\label{sec:method}

\subsection{System Architecture}
\label{sec:architecture}

The proposed SoC integrates three
subsystems~\cite{waterman2016riscv,palnitkar2003verilog}:

\begin{itemize}
  \item The \textbf{5-stage pipeline CPU core} with integrated branch predictor;
  \item An \textbf{instruction ROM} initialized from a binary program image at
    simulation or configuration time;
  \item A \textbf{data RAM} supporting byte, halfword, and word load/store with
    byte-enable logic.
\end{itemize}

The baseline CPU uses an identical pipeline and memory hierarchy but without the
predictor: all branches are resolved at the EX stage under a static not-taken
assumption. This structural equivalence enables a direct, single-variable
comparison between the two designs---any measured performance difference is
attributable solely to the predictor.

\subsection{Pipeline Stage Design}
\label{sec:modules}

\paragraph{Stage~1: Instruction Fetch (IF).}
The IF stage presents the current PC to the instruction ROM. The PC register
selects among three next-PC sources in priority order: (1)~EX-stage redirect on
misprediction or unconditional jump, (2)~ID-stage speculative branch target when
the predictor predicts taken, and (3)~sequential PC$+4$. The IF/ID pipeline
register supports both stall (hold current value) and flush (insert NOP bubble)
controls.

\paragraph{Stage~2: Instruction Decode (ID).}
The ID stage performs opcode decoding for the implemented instruction subset;
extracts source and destination register addresses; generates sign-extended
immediates for the active RV32I format types (I, S, B, J); and produces memory,
branch, and register-file control signals.
The branch target address is computed combinationally as
$\mathrm{PC}_\mathrm{ID} + \mathrm{imm}_{B}$.

Simultaneously, the branch predictor queries its tables using the ID-stage PC and
a conditional-branch flag, returning a speculative prediction and a GHR snapshot
within the same clock cycle.

\paragraph{Stage~3: Execute (EX).}
The EX stage implements the ALU for integer operations (ADD, SUB, OR, AND,
XOR, SLT, shifts), computes branch comparisons (BEQ, BNE, BLT, BGE, BLTU, BGEU),
and resolves unconditional jumps. The actual branch outcome and misprediction
signal are forwarded to both the PC selection logic and the predictor update port.

\paragraph{Stage~4: Memory Access (MEM).}
The MEM stage drives the data RAM interface for store operations and routes load
data back through the pipeline. Byte and halfword loads are sign- or
zero-extended to 32 bits according to the instruction type.

\paragraph{Stage~5: Write-Back (WB).}
The MEM/WB pipeline register forwards the result word and destination register
address to the register file write port, completing the five-stage execution loop.

\subsection{Hazard Detection Unit}
\label{sec:hazard}

The hazard control unit implements three resolution
policies~\cite{hennessy2017quantitative}:

\begin{enumerate}
  \item \textbf{Misprediction or unconditional-jump redirect (EX stage):}
    Flush two pipeline stages to discard the incorrectly prefetched instructions
    and redirect the PC to the correct target. This is the primary
    misprediction recovery path.

  \item \textbf{Load-use hazard:}
    When an EX-stage memory load targets a register that is immediately used
    as a source by the ID-stage instruction, stall the fetch and decode stages
    and insert one pipeline bubble to allow the load result to be written back
    before it is consumed. Register~x0 is excluded.

  \item \textbf{ID-stage predicted-taken speculative prefetch:}
    When the predictor asserts a taken outcome in the ID stage, flush the
    fall-through instruction already in the IF stage, since the PC has been
    redirected to the predicted branch target.
\end{enumerate}

\subsection{Branch Predictor Architecture}
\label{sec:bp-arch}

The predictor is parameterized with 16 entries per table (index width = 4 bits),
an 8-bit Global History Register, 4-bit signed weights in the range $[-8,+7]$,
and an 8-bit signed accumulator.

\paragraph{Bimodal Predictor.}
A 16-entry array of 2-bit saturating counters indexed by the lower bits of the
PC. Each counter initializes to weak not-taken on reset. The counter MSB drives
the prediction directly. Update rule: increment toward strong-taken if the branch
was taken, decrement toward strong-not-taken if not taken, with saturation at both
ends~\cite{smith1981study}.

\paragraph{Confidence Estimator.}
A single combinational gate that asserts high confidence only when the saturating
counter has reached a strong state (either $00_2$ or $11_2$). Weak states
($01_2$ or $10_2$) indicate a branch that has recently reversed direction and is
therefore a candidate for the neural path:
\[
  \text{high confidence} \iff c \;\in\; \{00_2,\; 11_2\}
\]

\paragraph{Global History Register.}
An 8-bit shift register updated on every resolved conditional branch: shift left,
insert the actual outcome into bit~0. The current GHR value is snapshotted into
a sideband pipeline register at the ID stage so that the EX-stage update uses
the history state that was active when the prediction was
made~\cite{yeh1991twolevel}.

\paragraph{Perceptron Predictor.}
Sixteen perceptrons indexed by the lower bits of the PC, each holding a bias
register and eight weight registers, all 4-bit signed. The prediction is the sign
of~\cite{jimenez2001perceptron}:
\[
  y = w_{\text{bias}} + \sum_{i=0}^{7}
      \begin{cases}
        +w_i & \text{if } h[i] = 1 \\
        -w_i & \text{if } h[i] = 0
      \end{cases}, \qquad
  \hat{t} = \mathbf{1}[y \geq 0]
\]
where $h[i]$ is the $i$-th bit of the GHR. The computation is implemented as a
parallel combinational adder tree with the early-stopping rule applied within
the combinational path (Algorithm~\ref{alg:earlystop}); the result is latched
into a 1-cycle output register to meet the 125\,MHz timing constraint.

\begin{algorithm}[htbp]
\caption{Perceptron inference with hardware early stopping. In the RTL
  implementation, the full combinational tree is computed in parallel and
  the result is registered after one clock cycle; the early-stop condition
  reduces switching activity within the combinational path.}
\label{alg:earlystop}
\begin{algorithmic}[1]
\Require bias $b$, weights $w[0..7]$, GHR $h[0..7]$, $W_{\max}=7$
\State $s \leftarrow b$; \ $\textit{stop} \leftarrow \textit{false}$
\For{$i \leftarrow 0$ \textbf{to} $7$}
  \If{$\neg\textit{stop}$}
    \State $s \leftarrow s + (h[i]=1 \;?\; +w[i] : -w[i])$
    \If{$|s| > (7 - i) \times W_{\max}$}
      \State $\textit{stop} \leftarrow \textit{true}$ \Comment{remaining terms cannot change sign}
    \EndIf
  \EndIf
\EndFor
\State \Return $\hat{t} \leftarrow (s \geq 0)$
\end{algorithmic}
\end{algorithm}

Online weight update (triggered on misprediction only):
\[
  w_j \;\leftarrow\; \mathrm{clamp}\!\left(
    w_j \;+\; \begin{cases}
      +1 & \text{if } t_{\mathrm{actual}} = h[j] \\
      -1 & \text{otherwise}
    \end{cases},\;
    [-8,\,+7]
  \right)
\]
Bias is updated similarly: $+1$ if taken, $-1$ if not-taken.

\paragraph{Predictor Orchestration.}
The final prediction selects between the two paths based on bimodal confidence:
\[
  \hat{t} =
  \begin{cases}
    \hat{t}_{\text{bimodal}} & \text{if bimodal confidence is high} \\
    \hat{t}_{\text{neural}}  & \text{otherwise}
  \end{cases}
\]
The neural path is activated only when the branch is a conditional instruction
\emph{and} bimodal confidence is low.

\subsection{Prototype Demonstration and Operational Flow}
\label{sec:demo}

The system operates as a fully self-contained, demonstrable prototype. The
complete operational flow from power-on to measurable output is as follows:

\begin{enumerate}
  \item \textbf{Program loading.} At startup, the instruction ROM is initialized
    from a binary program image. No external loader or software tool chain is
    required at runtime.

  \item \textbf{Speculative pipeline execution.} The CPU begins fetching from
    address 0. For each conditional branch encountered in the ID stage, the
    predictor immediately produces a speculative outcome: if taken, the PC is
    redirected to the branch target and the fall-through instruction is flushed.
    Execution continues speculatively on the predicted path.

  \item \textbf{EX-stage resolution and online update.} When the branch reaches
    the EX stage, the ALU resolves the actual outcome. If the prediction was
    correct, no corrective action is taken. If it was wrong, the hazard control
    unit issues a two-stage flush, the PC is corrected to the actual target, and
    the predictor receives an online weight update.

  \item \textbf{Automated metric collection.} The head-to-head comparison
    testbench monitors internal predictor signals cycle-by-cycle and, at program
    completion, automatically prints a metrics report: total branches,
    correct/incorrect predictions, accuracy, penalty cycles, and neural
    activation count.

  \item \textbf{Functional pass/fail verification.} The branch-stress functional
    testbench verifies that the final register values x3~$=$~10, x4~$=$~20,
    and x30~$=$~30 are correct---confirming that speculative execution produced
    arithmetically exact program results.
\end{enumerate}

This end-to-end flow, from instruction loading through speculative execution to
automated metric reporting, demonstrates that the system is a fully operational
prototype, not a simulation artifact.

\subsection{Data Flow and Operational Workflow}
\label{sec:dataflow}

On each rising clock edge, the PC advances (or holds if stalled). The instruction
ROM returns the instruction word combinationally. The IF/ID register captures both
instruction and PC. In the ID stage, the decoder extracts all control and data
signals while the branch predictor queries its tables combinationally; if the
predictor asserts taken, the PC is speculatively redirected and the IF stage is
flushed (one bubble). The ID/EX register and five sideband registers latch decoded
operands and predictor metadata simultaneously. In the EX stage, the ALU resolves
branch conditions; a misprediction triggers a two-stage flush and a PC redirect.
The EX/MEM and MEM/WB registers propagate the result to write-back, which feeds
the register file.

The predictor update path is triggered on every resolved conditional branch: the
bimodal counter is always updated, but perceptron weight updates are additionally
gated to mispredicted branches only, limiting weight changes to the cases where
the prediction was wrong.

\subsection{Verification and Implementation Results}
\label{sec:results}

\subsubsection{Functional Verification}

\paragraph{Baseline CPU.}
The baseline functional testbench monitors the write-back bus cycle by cycle and
compares each register write-back against a pre-computed expected value table.
A sequence of core ISA operations is exercised in dependency order: immediate
loads, register arithmetic (addition and subtraction), bitwise logic (OR, AND),
a memory load, a set-less-than comparison, and a conditional branch.
Every result is checked at the write-back stage for both value correctness and
pipeline timing, providing direct evidence of correct data hazard detection,
load-use stall insertion, and branch-flush recovery.

\paragraph{Innovation CPU.}
The hybrid functional testbench runs the branch-stress loop program to completion
and tracks the final values of three architectural sentinel registers whose
correct values depend on the loop executing the precise number of taken and
not-taken paths over all 20 iterations. The testbench passes if and only if all
three sentinels match their analytically determined expected values, confirming
that the hybrid predictor's speculative execution, GHR snapshots, mispredict
flushes, and weight updates preserve full architectural correctness across all
40 conditional branches.

\subsubsection{Multi-Workload Simulation Study}
\label{sec:multiworkload}

To assess whether the predictor's advantage is workload-specific or broadly
applicable, five additional programs were written and simulated in head-to-head
testbenches alongside the original branch-stress program. Each program stresses
a different branch-pattern archetype:

\begin{itemize}
  \item \textbf{Strongly-taken loop.} A 32-countdown loop with 31 taken
    followed by 1 not-taken branch. This is the worst case for an
    always-not-taken predictor (3.13\% accuracy) and the best case for
    a predictor that can quickly learn the dominant direction.
  \item \textbf{Nested loop.} A 5$\times$4 nested iteration producing inner
    and outer loop-back branches with different repetition counts, testing
    per-PC table indexing and history tracking under two simultaneously
    active branch PCs.
  \item \textbf{Alternating forward.} A 16-iteration loop with a forward
    conditional branch that skips an increment when the loop variable is odd,
    producing alternating taken/not-taken at the forward branch alongside a
    strongly-taken back branch.
  \item \textbf{NT-biased.} A loop combining a never-taken comparison branch
    (x1$\neq$x2 always, 16 iterations) with a strongly-taken back branch,
    testing whether high-confidence not-taken branches are correctly handled
    without neural escalation.
  \item \textbf{Long countdown loop.} A 32-iteration countdown with the same
    pattern as strongly-taken, exercising the predictor over a longer learning
    horizon (31 taken, 1 not-taken).
\end{itemize}

To enable a head-to-head accuracy comparison against real open-source designs,
behavioral predictor models for PicoRV32 (stall-to-resolve, no prediction),
Ibex~+~SBP, and VexRiscv~+~BHT were implemented in a unified Icarus Verilog
testbench and executed against all six branch-pattern
programs using the same instruction memory images as the primary simulations.
The behavioral models are faithful to the documented predictor mechanisms:
the Ibex SBP predicts taken for any branch whose signed offset is negative
(backward branch) and not-taken otherwise; the VexRiscv BHT maintains a
direct-mapped 16-entry table of 2-bit saturating counters indexed by
\texttt{PC[5:2]}, initialised to weakly-not-taken.
PicoRV32 is modelled as incurring a 2-cycle stall on every conditional branch,
corresponding to a zero-prediction strategy.

(parallel adder output register and bimodal/confidence/GHR snapshot registers)
introduced to achieve timing closure at 125\,MHz.
No DSP blocks are consumed, confirming that the weight-update and dot-product
paths are implemented entirely with LUT-level addition and signed comparison,
as intended by the low-bit-width quantization design.

Post-route timing analysis confirms full timing closure at 125\,MHz for both
SoC variants. The innovation SoC achieves a worst negative slack (WNS) of
$+0.006$\,ns with zero setup violations and zero hold violations.
Timing closure was achieved by replacing the sequential early-stop loop in
\texttt{bp\_perceptron} with a parallel tree adder followed by a 1-cycle
output register, and by adding pipeline registers for \texttt{bimodal\_taken},
\texttt{conf\_high}, and \texttt{ghr\_snapshot} in \texttt{bp\_top}. The
IF-stage PC feed to the predictor (\texttt{u\_bp.pc\_id}) was also updated
from \texttt{if\_id\_pc} to \texttt{if\_out\_pc} to present the IF-stage PC
one cycle ahead, matching the new pipeline timing.
Both the baseline and innovation SoC complete place-and-route successfully
with no routing errors; functional correctness is confirmed by behavioral
simulation at the register-transfer level.

The open-source Ibex core ($\ddagger$, out-of-context, no memory) synthesizes
to 2{,}331 LUTs and 876 FFs at 0.141\,W total / 0.038\,W dynamic under identical
tool and clock settings, confirming that the lowRISC Ibex pipeline (with full
compressed-instruction support, CSR registers, and PMP unit) is substantially
more complex than our purpose-built five-stage core, yet achieves comparable
average prediction accuracy (84.95\,\% vs.\  our 84.39\,\%) when its
static backward-taken / forward-not-taken heuristic is evaluated on the same
six-workload suite.

\section{Maximum Benefits of the Research Outcome}
\label{sec:benefits}

\subsection*{Benefit 1: Measured 78.31\% Average Reduction in Branch Misprediction Penalty Across Six Workloads}

The most directly quantifiable benefit is a \textbf{78.31\% average reduction in
approximate pipeline penalty cycles} across six distinct branch-pattern workloads,
with individual reductions ranging from 58.33\% (alternating-condition pattern) to
96.77\% (strongly-biased long loop). On the primary 40-branch stress program the
reduction is 65.52\%---from 58 penalty cycles (baseline) to 20---corresponding
to a prediction accuracy gain from 27.50\% to 75.00\%. On the long countdown loop,
accuracy reaches 96.88\% versus a baseline of 3.13\%, eliminating virtually all
mispredictions without any ISA change or binary modification.

The improvement is structurally consistent across all six archetypes: strongly-biased
branches are caught by the bimodal table after one or two observations; alternating
branches are escalated to the perceptron path where weight adaptation delivers
above-random accuracy that improves with each successive iteration; and mixed
NT-biased workloads benefit from confidence gating that routes the never-taken
branch through the bimodal path at high confidence, reserving neural inference only
for genuinely uncertain branches. In real embedded workloads with higher branch
densities (nested loops, interrupt handlers, recursive algorithms), reducing flush
overhead translates directly into increased effective instructions-per-cycle~\cite{hennessy2017quantitative}.

\subsection*{Benefit 2: Neural Inference Cost Proportional to Branch Difficulty}

The confidence-adaptive architecture achieves neural accuracy gains without paying
full neural inference cost on every branch. Two complementary mechanisms deliver
this:

\begin{itemize}
  \item \textbf{Confidence gating:} 45\% of branches are resolved entirely within
    the bimodal path, requiring no weight-table reads and no dot-product arithmetic.
    Neural activation is limited to the 55\% of branches where bimodal confidence
    is genuinely low. On the 40-branch stress program, only 22 of 40 branches
    invoke the perceptron, reducing neural switching activity by 45\% versus
    an always-on perceptron.

  \item \textbf{Early stopping:} Within the perceptron combinational path, a
    magnitude-bound comparator halts the accumulation tree as soon as the partial
    sum sign is mathematically determined, reducing LUT switching activity for
    early-convergent branches. In the pipelined RTL implementation, the output
    register fixes the observable latency at one clock cycle; the benefit is
    reduced dynamic toggling within the combinational cone rather than a
    variable output latency.
\end{itemize}

Crucially, the entire arithmetic path uses 4-bit integer operations (add,
subtract, and magnitude comparison) with no multiply-accumulate DSP blocks;
all logic maps to LUT fabric. This design makes the predictor viable for entry-level FPGA devices and
standard-cell ASIC flows where DSP resources are shared, and reduces dynamic
toggling relative to an always-active full-precision neural approach.

\subsection*{Benefit 3: Fully Verified, Reusable RTL Platform for RISC-V Development}

The complete system is delivered as a clean, parameterized hierarchy of synthesizable
Verilog modules with a ten-testbench verification suite and a head-to-head
comparative evaluation framework spanning six distinct branch-pattern workloads. The comparison testbench automatically prints a
complete metrics report at simulation end---accuracy, misprediction count, and
neural activation count---requiring no manual waveform inspection. Changing the predictor table size, history length, or weight bit-width
is sufficient to explore the full accuracy-versus-area trade-off space without
modifying any pipeline logic.

FPGA validation on PYNQ-Z2 at 125\,MHz confirms that the system is a physically
realizable prototype, not a simulation artifact. This establishes the design as
a directly usable foundation for: extended SoC integration (peripheral buses,
interrupt controllers, RTOS deployment); computer architecture education platforms
where a working adaptive predictor is more instructive than a static one; and
pre-silicon branch-predictor evaluation before ASIC tape-out on an open-source PDK.
The absence of FPGA-specific primitives ensures full portability to other device
families and standard-cell flows.

\section{Conclusion and Future Prospects}
\label{sec:conclusion}

This project has designed, implemented, and validated a complete 5-stage
pipelined RISC-V RV32I SoC featuring a confidence-adaptive hybrid neural branch
predictor, realized as synthesizable Verilog RTL and confirmed on the PYNQ-Z2
FPGA at 125\,MHz. The delivered system includes a fully operational CPU core,
instruction ROM, data RAM, hazard detection unit, and a self-contained predictor
plugin with five sideband pipeline registers providing precise update semantics.

The results are concrete and measured: across a six-program evaluation
suite spanning strongly-biased loops, nested loops, alternating patterns, and
mixed-bias workloads, the hybrid predictor achieves an average accuracy of
\textbf{84.39\%} versus \textbf{23.87\%} for the always-not-taken baseline,
delivering an average \textbf{78.31\% reduction} in misprediction penalty
cycles. On the primary 40-branch stress program, accuracy reaches 75.00\%
with a 65.52\% penalty reduction; on a 32-branch strongly-biased loop, accuracy
reaches 96.88\% with a 96.77\% penalty reduction. Confidence gating limits
neural activation to 55\% of branches, all within a LUT-only implementation
requiring no DSP blocks. Both CPU variants achieve full place-and-route timing
closure at 125\,MHz on the PYNQ-Z2 (WNS\,=\,+0.006\,ns, zero violations),
enabled by a pipelined parallel-tree adder in the perceptron inference path.

The following directions are planned for future development:

\begin{itemize}
  \item \textbf{Bus interconnect and peripheral integration.}
    An AXI-Lite / APB bus interconnect, followed by peripheral modules (UART, SPI,
    I\textsuperscript{2}C, DMA controller, Ethernet MAC), and a PLIC/CLINT interrupt
    controller compliant with the RISC-V Privileged ISA for RTOS deployment.

  \item \textbf{Operand forwarding.}
    Adding EX$\rightarrow$EX and MEM$\rightarrow$EX forwarding paths would
    eliminate most load-use stall cycles and further improve effective IPC.

  \item \textbf{Predictor parameter exploration.}
    The parameterized architecture enables systematic studies across table sizes,
    history lengths, and weight bit-widths on a wider suite of benchmark programs
    beyond the branch-stress microbenchmark.

  \item \textbf{ASIC tape-out.}
    The synthesizable coding style (no FPGA-specific primitives, no block-RAM
    macros, no DSP instantiations) makes the design directly portable to
    standard-cell ASIC flows for silicon-level PPA validation.
\end{itemize}

This work demonstrates that a meaningful, measured improvement in branch prediction
quality can be achieved on resource-constrained hardware through a carefully
engineered integration of complementary prediction mechanisms---without
multiply-accumulate hardware, large on-chip memories, or proprietary IP. The
resulting platform is a solid, verifiable, and openly extensible foundation for
continued RISC-V SoC development.
